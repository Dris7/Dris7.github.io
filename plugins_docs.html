Overview

The Moflex Plugin System enables seamless integration of various streaming sources into the Moflex platform. This documentation provides comprehensive guidance on creating and implementing plugins that enhance the streaming experience while maintaining compatibility with the core platform.

Key Features

  • Standardized metadata structure for consistent integration
  • Flexible stream type support (API or WebView)
  • Built-in support for multiple quality options
  • Subtitle integration capabilities
  • Responsive video player integration

Getting Started

Before implementing a plugin, ensure you have:

  • A valid streaming source with stable API endpoints
  • Capability to serve JSON responses in the required format
  • Understanding of the metadata structure and requirements

Metadata Structure

The plugin metadata defines how your streaming source integrates with Moflex. Here's a complete example:

{
  "name": "ExamplePlugin",
  "embed_url": "https://example.com/embed/{t}/{i}",
  "tv_embed_url": "https://example.com/embed/{t}/{i}/{s}/{e}",
  "stream_type": "api",
}

Detailed Field Reference

Field Required Description
name Yes Unique identifier for your plugin
embed_url Yes Template URL for movie content
tv_embed_url Yes Template URL for TV show content
stream_type Yes Either "api" or "webview"

API Response Structure

When using stream_type: "api", your endpoint must return this JSON structure:

{
  "qualities": [
    {
      "name": "1080p",
      "link": "https://example.com/beautiful.mp4",
      "bitrate": "5000k",
      "resolution": "1920x1080"
    },
    {
      "name": "Auto",
      "link": "https://example.com/master.m3u8",
      "type": "hls"
    }
  ],
  "subtitles": [
    {
      "name": "English",
      "link": "https://example.com/en.vtt",
      "language": "en",
      "type": "vtt"
    },
    {
      "name": "Spanish",
      "link": "https://example.com/es.srt",
      "language": "es",

  ],
  "metadata": {
    "title": "Example Movie",
    "duration": 7200,
    "thumbnail": "https://example.com/thumb.jpg"
  }
}

Best Practices

Recommended Implementation

  • Always implement error handling in your API responses
  • Use HTTPS for all URLs
  • Implement proper CORS headers
  • Cache responses when possible
  • Provide fallback quality options

Error Response Example

{
  "error": true,
  "code": "CONTENT_UNAVAILABLE",
  "message": "The requested content is not available",
  "status": 404
}
